# 智慧医疗系统数据库设计文档

> **已知缺口**：本设计文档当前未收录 `audit_log`、`login_log`、`sys_role` / `sys_permission` / `sys_role_permission` / `sys_user_role`、`sys_service_account`、`prescription` / `prescription_item`、`refund_order`、`symptom_rule` / `high_risk_symptom_rule` / `negative_rule`、`ai_report_text_analysis` / `ai_image_detection` 等表。它们的字段定义见 `docs/修改建议.md`（§2.1 处方、§2.2 审计、§2.3 RBAC、§3.1 症状规则、§3.2 影像拆分、§6.2 退费）。代码实现时各服务 `schema.sql` 为准，本设计文档将在对应表落地后回填。实施状态总览见 `docs/修改建议.md` 第一节"实施状态"列与 `docs/遗留问题复盘与实施状态.md`。

## 1. 设计说明

本文档按业务边界划分为两部分：传统业务表和 AI 业务表。结构化数据必须使用 MySQL 8.0 存储，并通过 MyBatis-Plus 访问；疾病 JSON 派生出的文本向量必须使用 Milvus 或 Qdrant 存储，用于 RAG 检索。

通用约定：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键，自增或雪花 ID |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| deleted | TINYINT | 逻辑删除标识，0 未删除，1 已删除 |

技术约束：

| 类型 | 必选技术 | 说明 |
| --- | --- | --- |
| 结构化业务数据 | MySQL 8.0 | 保存传统业务表和 AI 业务结果表 |
| 数据访问 | MyBatis-Plus | 用于 CRUD、分页、条件查询和代码生成 |
| 缓存与锁 | Redis | 缓存热点数据、验证码、限流和分布式锁 |
| 异步消息 | RabbitMQ | 解耦 AI 任务、通知投递和调用日志写入 |
| 向量数据 | Milvus 或 Qdrant | 保存疾病 JSON 切分文本的 Embedding |

## 3. AI 业务表

### 3.1 disease_json_knowledge 疾病 JSON 知识数据结构

**表作用**：描述 RAG 检索使用的疾病 JSON 数据结构。该数据只约束 JSON 字段含义，不约束具体存储介质或访问方式。

**向量化说明**：疾病 JSON 中的 `name`、`desc`、`cause`、`symptom`、`acompany`、`cure_department`、`check` 等字段可切分为文本片段并生成 Embedding，存入 Milvus 或 Qdrant。向量库只保存检索索引和片段元数据，不作为新的医学知识来源。

| 字段名 | 数据类型 | 说明 |
| --- | --- | --- |
| source_id | VARCHAR(100) | JSON 数据唯一标识，可使用原始 `_id` 或业务生成 ID |
| name | VARCHAR(200) | 疾病名称 |
| desc | TEXT | 疾病描述 |
| category | JSON | 疾病分类 |
| prevent | TEXT | 预防方式 |
| cause | TEXT | 病因 |
| symptom | JSON | 症状列表 |
| yibao_status | VARCHAR(20) | 医保状态 |
| get_prob | VARCHAR(50) | 患病概率 |
| get_way | VARCHAR(200) | 传播方式 |
| acompany | JSON | 并发症 |
| cure_department | JSON | 推荐科室 |
| cure_way | JSON | 治疗方式 |
| cure_lasttime | VARCHAR(100) | 治疗周期 |
| cured_prob | VARCHAR(50) | 治愈概率 |
| cost_money | VARCHAR(200) | 费用说明 |
| check | JSON | 检查项 |
| recommand_drug | JSON | 推荐药物 |
| drug_detail | JSON | 药物详情 |

### 3.2 disease_vector_index 疾病向量索引结构

**表作用**：描述 Milvus 或 Qdrant 中疾病 JSON 向量索引的字段结构，用于 RAG 相似度检索。该结构不是 MySQL 业务表。

| 字段名 | 数据类型 | 说明 |
| --- | --- | --- |
| vector_id | VARCHAR(100) | 向量 ID |

| metadata | JSON | 元数据，如分类、推荐科室、更新时间 |
