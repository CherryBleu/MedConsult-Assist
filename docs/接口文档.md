# 智慧医疗系统接口文档

## 1. 接口约定

| 项目 | 内容 |
| --- | --- |
| Base URL | `/api/v1` |
| 数据格式 | `application/json` |
| 认证方式 | `Authorization: Bearer <access_token>` |
| 时间格式 | ISO 8601，例如 `2026-07-06T10:30:00+08:00` |
| 分页参数 | `page` 从 1 开始，`pageSize` 默认 10 |

**技术实现约束**

| 类别 | 技术要求 |
| --- | --- |
| 基础框架 | JDK 21、Spring Boot 3.x |
| 微服务 | Spring Cloud Alibaba：Nacos、Gateway、OpenFeign |
| 持久层 | MyBatis-Plus、MySQL 8.0 |
| 缓存与消息 | Redis、RabbitMQ |
| AI 接入 | Spring AI 或 LangChain4j，适用于智能分诊、病历摘要、用药分析、影像检测等大模型能力；症状自诊问答以本地 Embedding 向量检索 + 规则引擎为核心，可叠加生成式 LLM 做口语化输入标准化与回答润色，回答须可追溯到疾病 JSON 命中片段 |
| 向量检索 | Milvus 或 Qdrant |
| 前端与部署 | Vue 3、Docker、GitLab CI/CD |

说明：RAG 的原始知识来源只允许疾病 JSON，Milvus 或 Qdrant 只保存疾病 JSON 文本切分后的向量索引。症状自诊问答以本地 Embedding 模型（如 bge-small-zh-v1.5）向量召回 + 症状抽取规则 + 高危症状判定规则 + 回答模板为核心链路；当规则难以覆盖口语化输入或需自然语言润色时，允许叠加生成式 LLM 做输入标准化与回答重组，但回答文本必须能追溯到疾病 JSON 命中片段，不得引入模型自由生成的医学论断；其他 AI 功能通过 Spring AI 或 LangChain4j 接入生成式大模型。

**统一响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "trace-001",
  "timestamp": "2026-07-06T10:30:00+08:00"
}
```

**通用错误码**

| code | HTTP | 说明 |
| --- | --- | --- |
| 0 | 200 | 成功 |
| 400001 | 400 | 请求参数错误 |
| 401001 | 401 | 未登录或 Token 失效 |
| 403001 | 403 | 无访问权限 |
| 404001 | 404 | 资源不存在 |
| 409001 | 409 | 业务冲突 |
| 500001 | 500 | 系统内部错误 |
| 502001 | 502 | 外部 AI 服务调用失败 |

#### 2.1.6 管理员查询用户列表

`GET /auth/users?page=1&pageSize=20&keyword=张三&role=PATIENT`

> 内部扩展接口（RBAC 列表查询）。**鉴权：仅 `HOSPITAL_ADMIN` 可访问**，service 层手动校验角色，非管理员返回 403 FORBIDDEN。不含 passwordHash 字段，避免密码摘要泄露。

**请求参数（Query）**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page` | int | 否 | 页码，从 1 开始，默认 1 |
| `pageSize` | int | 否 | 每页条数，默认 20，上限 100 |
| `keyword` | string | 否 | 按账号/姓名/手机号模糊搜索 |
| `role` | string | 否 | 角色过滤：`PATIENT` / `DOCTOR` / `PHARMACY_ADMIN` / `HOSPITAL_ADMIN` |

**返回示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "page": 1,
    "pageSize": 20,
    "total": 1,
    "items": [
      {
        "id": 1001,
        "userNo": "U202607060001",
        "account": "zhangsan",
        "phone": "138****0001",
        "name": "张三",
        "role": "PATIENT",
        "patientId": 5001,
        "doctorId": null,
        "pharmacistId": null,
        "status": "ACTIVE",
        "createdAt": "2026-07-06T09:00:00"
      }
    ]
  },
  "traceId": "trace-auth-users",
  "timestamp": "2026-07-06T11:00:00+08:00"
}
```

**返回字段（`UserListItem`）**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | Long | 用户主键 ID |
| `userNo` | String | 用户业务编号 |
| `account` | String | 登录账号 |
| `phone` | String | 手机号（脱敏） |
| `name` | String | 姓名 |
| `role` | String | 角色（来自 Redis，兜底 `PATIENT`） |
| `patientId` | Long | 关联患者档案 ID（无则 null） |
| `doctorId` | Long | 关联医生 ID（无则 null） |
| `pharmacistId` | Long | 关联药师 ID（无则 null） |
| `status` | String | 状态：`ACTIVE` / `DISABLED` / `LOCKED` |
| `createdAt` | LocalDateTime | 创建时间 |

#### 2.1.7 修改密码

`POST /auth/change-password`

> 当前登录用户修改自己的密码（#19）。**鉴权：需登录**（任意角色，改自己密码，无需管理员）。校验原密码正确后用 BCrypt 重新编码新密码。改密后旧 access token 仍有效至过期（当前无服务端 access token 失效机制），前端应提示用户重新登录。

**请求体（`ChangePasswordRequest`）**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `oldPassword` | String | 是 | 原密码 |
| `newPassword` | String | 是 | 新密码：8-64 位，至少含字母和数字（同注册强度） |

**请求示例**

```json
{ "oldPassword": "Old12345", "newPassword": "New12345" }
```

**返回**：`Result<Void>`（`code:0` 成功，无 data）。原密码错误返回 400 `PARAM_ERROR`，新密码与原密码相同返回 400 `PARAM_ERROR`。

#### 2.1.8 管理员创建用户

`POST /auth/users`

> 管理员后台创建用户（#20/#21）。**鉴权：仅 `HOSPITAL_ADMIN` 可访问**，service 层手动校验角色。区别于自助 `POST /auth/register`：支持创建所有角色（含 `PHARMACY_ADMIN` / `HOSPITAL_ADMIN`），role 校验用 `ALLOWED_ROLES`。PATIENT 角色仍会自动建档（Feign 调 patient-service），其余角色仅建账号。建号后写 Redis 角色 key，否则登录角色兜底 `PATIENT` 导致跳转异常。

**请求体（复用 `RegisterRequest`）**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | String | 是 | 账号：4-32 位字母/数字/下划线 |
| `password` | String | 是 | 密码：8-64 位，至少含字母和数字 |
| `phone` | String | PATIENT 必填 | 手机号（11 位） |
| `name` | String | 是 | 姓名：1-50 位中文/字母 |
| `role` | String | 否 | `PATIENT`/`DOCTOR`/`PHARMACY_ADMIN`/`HOSPITAL_ADMIN`，默认 `PATIENT` |
| `idCard` | String | PATIENT 必填 | 身份证号（建档用） |
| `patientId`/`doctorId` | String | 否 | 关联已有档案编号 |

**返回示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": "U202607160001",
    "name": "新管理员",
    "role": "HOSPITAL_ADMIN",
    "patientId": null, "doctorId": null, "pharmacistId": null,
    "status": "ACTIVE"
  }
}
```

**返回字段（`UserInfo`）**：`userId` / `name` / `role` / `patientId` / `doctorId` / `pharmacistId` / `status`。账号/手机号重复返回 409 `CONFLICT`。

#### 2.3.1b 新增科室

`POST /departments`

> 管理员新增科室（#15）。**鉴权：仅 `HOSPITAL_ADMIN`**（`@Permission` 切面）。`departmentNo` 由后端自动生成（`DEP_` + 雪花 ID base36），前端不传。名称重复返回 409 `CONFLICT`。

**请求体（`CreateRequest`）**：`name`(必填) / `description` / `location` / `enabled`(Boolean，默认 true)。

**返回**：`{ "departmentId": "DEP_..." }`（`MutationResponse`）。

#### 2.3.1c 更新科室

`PATCH /departments/{departmentId}`

> 更新科室部分字段（#15）。**鉴权：仅 `HOSPITAL_ADMIN`**。全部字段可选，未传保留原值。`departmentId` 为科室编号（department_no）。名称变更时校验唯一性（排除自身）。

**请求体（`UpdateRequest`）**：`name` / `description` / `location` / `enabled`(Boolean) 任选。

**返回**：`{ "departmentId": "DEP_..." }`。科室不存在返回 404 `NOT_FOUND`。

#### 2.3.1d 删除科室

`DELETE /departments/{departmentId}`

> 逻辑删除科室（#15）。**鉴权：仅 `HOSPITAL_ADMIN`**。删除前校验是否被医生/排班引用，被引用时返回 409 `CONFLICT` 拒绝删除（避免破坏医生管理/排班/分诊链路）。种子科室（心内科等，被 ai-service 分诊硬编码引用）不可删。

**返回**：`Result<Void>`（`code:0` 成功）。

#### 2.4.5 更新排班（全量）

`PUT /schedules/{scheduleId}`

**请求体（`UpdateRequest`）**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |

### 2.6b 处方管理

> 处方独立成表后的 8 态状态机接口（对齐《修改建议》§2.1）。路径前缀 `/api/v1/prescriptions`，走 Gateway 鉴权；路径变量 `prescriptionId` 实为 `prescription_no`（业务可读编号）。权限标注见《修改建议》§2.3 权限矩阵：开方/提交=DOCTOR，审方/调剂/完成=PHARMACY_ADMIN，缴费=PATIENT，列表/详情=DOCTOR+PHARMACY_ADMIN+HOSPITAL_ADMIN，退方=DOCTOR+PHARMACY_ADMIN。

**处方状态机（8 态）：**
```
DRAFT ──提交──▶ PENDING_REVIEW ──通过──▶ APPROVED ──缴费──▶ PAID
                       │                     │
                     驳回                  退方
                       ▼                     ▼
                   REJECTED             CANCELLED
APPROVED/PAID ──调剂──▶ DISPENSED ──完成──▶ COMPLETED
```

#### 2.6b.1 创建处方（开方）

`POST /prescriptions` —— `@Permission(roles={DOCTOR})`

请求体：
```json
{
  "recordId": "MR202607060001",
  "patientId": "PATIENT_NO",
  "doctorId": "DOCTOR_NO",
  "departmentId": "DEP_XXX",
  "source": "OUTPATIENT",
  "items": [
    { "drugNo": "DRUG_NO", "drugName": "硝苯地平", "specification": "30mg",
      "dosage": "1片", "frequency": "qd", "route": "口服", "days": 7,
      "quantity": 7, "unit": "片", "unitPrice": 2.50 }
  ]
}
```
响应：
```json
{
  "code": 0, "data": { "prescriptionId": "RX...", "status": "DRAFT", "totalFee": 17.50 },
  "traceId": "...", "timestamp": "..."
}
```

#### 2.6b.2 分页查询处方

`GET /prescriptions?page=1&pageSize=10&status=PENDING_REVIEW` —— `@Permission(roles={DOCTOR,PHARMACY_ADMIN,HOSPITAL_ADMIN})`

响应项 `ListItem`：`{ prescriptionId, recordId, status, totalFee, paymentStatus, createdAt }`

#### 2.6b.3 查询处方详情

`GET /prescriptions/{prescriptionId}` —— `@Permission(roles={DOCTOR,PHARMACY_ADMIN,HOSPITAL_ADMIN})`

响应 `DetailResponse`：`{ prescriptionId, status, source, totalFee, paymentStatus, pharmacyPharmacistId, reviewedAt, reviewComment, rejectReason, createdAt, items[] }`，`items[]` 含 `{ drugName, specification, dosage, frequency, route, days, quantity, unit, unitPrice, subtotal }`

#### 2.6b.4 提交审方

`POST /prescriptions/{prescriptionId}/submit` —— `@Permission(roles={DOCTOR})`，DRAFT → PENDING_REVIEW。响应 `{ prescriptionId, status }`

#### 2.6b.5 药师审方

`POST /prescriptions/{prescriptionId}/review` —— `@Permission(roles={PHARMACY_ADMIN})`，PENDING_REVIEW → APPROVED | REJECTED（Redis 锁内防并发）。

请求体：`{ "action": "APPROVE|REJECT", "pharmacistId": 123, "reviewComment": "...", "rejectReason": "..." }`

#### 2.6b.6 处方缴费

`POST /prescriptions/{prescriptionId}/pay` —— `@Permission(roles={PATIENT})`，APPROVED → PAID（Redis 锁内）。

#### 2.6b.7 调剂发药

`POST /prescriptions/{prescriptionId}/dispense` —— `@Permission(roles={PHARMACY_ADMIN})`，APPROVED/PAID → DISPENSED（同步 Feign 调 drug-service FEFO 出库，锁租约 20s；任一明细失败反向补偿回滚）。

#### 2.6b.8 发药完成

`POST /prescriptions/{prescriptionId}/complete` —— `@Permission(roles={PHARMACY_ADMIN})`，DISPENSED → COMPLETED。

#### 2.6b.9 取消处方（退方）

`POST /prescriptions/{prescriptionId}/cancel` —— `@Permission(roles={DOCTOR,PHARMACY_ADMIN})`，APPROVED/PAID → CANCELLED。请求体：`{ "reason": "..." }`
