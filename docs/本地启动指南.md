# 本地启动指南（端到端环境）

> 面向不熟悉环境配置的同事。目标：从零把整套系统跑起来，能用真实数据走业务流。
>
> **前置条件（你的机器已有）**：JDK 21 ✅、Node.js ✅（前端能跑）。
> **唯一需要你装的**：Docker Desktop。

## 整体架构（先理解）

```
你的浏览器（前端 3000）
    ↓
gateway:8080（网关，按路径转发到下游服务）
    ↓ Nacos 服务发现
┌─────────────────────────────────────────────┐
│ auth:8081  patient:8082  outpatient:8083    │
│ medical-record:8084  drug:8085  ai:8086     │
│ notification:8087                           │
└─────────────────────────────────────────────┘
    ↓                          ↓
中间件容器（docker-compose）   ai 外部依赖
MySQL/Redis/RabbitMQ/Nacos     MongoDB/LLM/Milvus/Embedding
```

## 启动顺序（4 步）

### 第 1 步：安装 Docker Desktop（只做一次）

1. 下载：https://www.docker.com/products/docker-desktop/
2. 双击安装 → 重启电脑（可能需要 BIOS 开虚拟化，一般默认开）
3. 打开 Docker Desktop，等左下角变**绿色 "Engine running"**
4. 验证：打开命令行（Git Bash 或 PowerShell）跑 `docker --version`，能输出版本号即可

### 第 2 步：启动中间件（一键）

在仓库根目录，依次跑（我会帮你跑，你只需确认 Docker 已装好）：

```bash
# 首次：复制环境变量模板
cp infra/.env.example infra/.env

# 启动全部中间件（首次拉镜像约 5-10 分钟，之后秒起）
docker compose -f infra/docker-compose.yml up -d

# 看健康状态（等所有服务变 healthy）
docker compose -f infra/docker-compose.yml ps
```

启动的容器（端口都是宿主机映射）：
- MySQL 8.0 → `127.0.0.1:3307`（root/root123456，业务账号 medconsult/medconsult123）
- Redis 7 → `127.0.0.1:16379`（无密码）
- RabbitMQ → `127.0.0.1:5672`（medconsult/medconsult123）
- **Nacos → `127.0.0.1:8848`**（控制台 http://localhost:8848/nacos，冒烟关了鉴权）
- Milvus / MinIO / Embedding（ai-service 用，见下"ai 缺口"）

验证 Nacos 起来了：浏览器开 http://localhost:8848/nacos（能看到页面就 OK）。

### 第 3 步：启动后端 7+1 个服务（我来跑）

**重要**：服务依赖中间件在线，必须第 2 步完成后再起。启动顺序：

| 批次 | 服务 | 端口 | 说明 |
|---|---|---|---|
| 1 | auth-service | 8081 | 登录/RBAC，最先起 |
| 1 | patient-service | 8082 | 患者档案（含 id_no 加密） |
| 1 | outpatient-service | 8083 | 科室/医生/排班/预约 |
| 1 | drug-service | 8085 | 药品/库存 |
| 1 | medical-record-service | 8084 | 病历/处方 |
| 1 | notification-service | 8087 | 通知/审计 |
| 2 | ai-service | 8086 | **依赖 MongoDB + LLM Key**（见下） |
| 3 | gateway | 8080 | 网关，最后起（聚合路由） |

启动命令（在 `backend/` 目录，每个开一个终端窗口）：
```bash
mvn spring-boot:run -pl auth-service
# 或用 IDE（IntelliJ）跑各服务的 Application 主类，更直观
```

每个服务启动约 30-60 秒。看到 `Started XxxApplication in xx seconds` 就是起来了。
首次启动各服务的 schema.sql 会自动建表 + DataSeeder 种子（admin/doctor/patient 账号，密码均 123456）。

**ai-service 特殊**：它依赖：
- MongoDB（`mongodb://localhost:27017`）——**本机没装会起不来**，需要单独装 MongoDB Community Server
- 阿里云百炼 LLM API Key——在 `infra/.env` 填 `ALIYUNBAILIAN_APIKEY=sk-xxx`（要去阿里云控制台申请）
- Milvus + Embedding（compose 已带，OK）

**如果你只想测核心业务（不测 AI）**：跳过 ai-service，其他 7 个服务能完整跑通登录/建档/预约/排班/开方/审方/退费/审计/RBAC。AI 链路（症状自诊/分诊/摘要/影像）等补齐 MongoDB + LLM Key 再说。

### 第 4 步：启动前端 + 测

```bash
cd frontend
# .env.development 里 VITE_USE_MOCK 改成 false（切换真实后端）
npm run dev
```

前端跑在 http://localhost:3000，调 http://localhost:8080（gateway）。
用 admin/doctor/patient/pharmacy 登录（密码都是 123456）。

## 常见问题

### Q: 某服务启动报 "Connection refused 127.0.0.1:3307"
A: MySQL 容器没起来或还没 healthy。`docker compose -f infra/docker-compose.yml ps` 看状态，等 MySQL 变 healthy 再起服务。

### Q: gateway 转发返回 503/找不到服务
A: Nacos 没起来，或下游服务还没注册到 Nacos。服务启动后约 10-30 秒才注册，稍等。http://localhost:8848/nacos → 服务管理 → 服务列表，应看到 auth-service/patient-service 等实例。

### Q: ai-service 启动失败
A: 检查 MongoDB（localhost:27017）是否在线、`.env` 里 ALIYUNBAILIAN_APIKEY 是否填了有效 key。缺这两个 ai 起不来。

### Q: 端口冲突（8080/8081 等被占）
A: `netstat -ano | findstr :8080` 找占用进程，关掉；或改对应服务 application.yml 的 port。
